### Database Schema Plan

#### User System
```sql
- users (id, email, password_hash, first_name, last_name, avatar, created_at, updated_at)
- user_profiles (user_id, bio, preferences, notification_settings)
- user_sessions (user_id, token, expires_at)
```

#### Team System
```sql
- teams (id, name, description, created_by, created_at)
- team_members (team_id, user_id, role, joined_at)
- team_invitations (team_id, email, token, expires_at)
```

#### Development Tracking
```sql
- goals (id, team_id, user_id, title, description, target_date, status)
- progress_entries (id, goal_id, user_id, progress_value, notes, created_at)
- activities (id, user_id, team_id, activity_type, description, created_at)
```

#### Document Management
```sql
- documents (id, team_id, user_id, filename, file_path, file_type, size, created_at)
- document_shares (document_id, shared_with_user_id, permissions)
```

## Future Deployment Plan (Live Website)

### Hosting: Railway 🚂
- Connect GitHub repo for auto-deploy on every push
- Natively supports Django with minimal config
- Can host PostgreSQL database in same dashboard
- Usage-based pricing: typically $5–15/month for small apps

### Database: PostgreSQL (via Railway)
- Perfect for Django with user accounts, tiers, relational data
- One-click provisioning in Railway
- Connection string drops directly into Django settings.py

### Full Deployment Process:
1. Build React app → `npm run build` → outputs `/build` folder
2. Django serves build folder as static files (natively)
3. Push everything to one GitHub repo
4. Connect repo to Railway → auto-detects Django
5. Add Postgres database with one click → paste connection URL
6. Every `git push` to main → Railway redeploys automatically

**Result**: Frontend, backend, database all in one Railway project. One dashboard, one bill.
